# 17143번. 낚시왕

## 문제
https://www.acmicpc.net/problem/17143

---

## Key Points

1. 문제를 풀기 위해 어떤 자료구조가 필요할지 생각해보면 좋다.

    - 하나의 열씩 이동하면서 낚시하는 것을 구현하기 위해 보드를 `board: list[list[int]]` 형태로 구성하여, 각 좌표에 위치한 상어의 idx를 표시하였다.

    - 낚이거나 잡아먹힌 상어를 효율적으로 제거해주기 위해 상어를 `sharks: dict[int, tuple[int, int, int, int, int]] = {idx: (x, y, s, d, z)}` 형태로 저장하였다.
        - 딕셔너리를 사용하면, 상어의 idx를 기반으로 O(1)에 제거 가능하다.
        - sharks에 저장할 때 `x`, `y`, `d`의 경우 1-based => 0-based로 변환하여 저장하였다. (-1 했다는 의미)

2. 자료구조를 잘 정했다면, 그 다음부터는 문제에서 요구하는 대로 구현하면 되는 문제이다.

    - 오른쪽으로 이동하면서 가장 작은 행의 상어를 찾아 낚는다.

    - 상어를 이동시킨다. 이동 후 해당 좌표에 상어가 있으면 크기가 큰 상어를 board에 남겨 놓고 작은 상어는 제거 목록(`removed_sharks`)에 저장한다.
        - 동시에 이동하는 것을 구현하기 위해 `board`를 초기화해준다.

    - 제거 목록에 있는 상어를 `sharks`에서 제거한다. 

3. 시간 최적화에 대해 생각해봐야 하는 문제이다.

    - 문제 그대로 구현한 버전: [[풀이 - solve_naive.py](./solve_naive.py)]
        - 열의 개수(c <= 100) x 상어 개수(M <= 10,000)만큼 for문을 도는데, 상어의 이동을 `for _ in range(s)`로 구현하면 s가 최대 값이 1000이라서 시간이 초과될 수 있다.
        - 실제로 백준에 돌려봤을 때, PyPy3에서는 통과하지만 Python3에서는 시간 초과가 발생한다.

    - 상어의 이동을 수학적으로 계산한 버전: [[풀이 - solve.py](./solve.py)]
        - `sharks`에 값을 저장할 때 s를 `2 * (r - 1)` 또는 `2 * (c - 1)`로 나눈 나머지로 저장하여, s의 크기를 줄여 최적화할 수 있다.
        - `for _ in range(s)`로 매초 마다 상어의 좌표를 시뮬레이션 하는 것이 아닌, 상어의 좌표를 바로 계산해서 최적화할 수 있다.
        - 두 가지 최적화를 적용한 후, 최적화 전 코드와 비교했을 때 PyPy3에서 실행 시간이 1/20로 줄었으며, Python3에서도 시간초과 없이 충분히 통과한다.
