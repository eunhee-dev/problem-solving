# 17406번. 배열 돌리기 4

## 문제
https://www.acmicpc.net/problem/17406

---

## Key Points

1. 전형적인 백트래킹 문제이다.

    - `itertools.permutations`을 사용하면 더 간결한 구현이 가능하겠지만, 이전까지의 회전 연산을 재사용할 수 없어 비효율적이다. (회전 연산은 O(N * M)인데, 이 문제의 경우 N, M이 크지 않아 통과할 것 같다.) 

    - 백트래킹을 사용하면, 이전 단계까지의 회전 연산 상태를 그대로 이어받아 다음 연산을 적용하면 되기 때문에 더 효율적이다.

2. 회전 연산 구현 시 인덱스 실수에 유의하여야 한다.

    - 헷갈리지 않도록 자신만의 기준(예: 가장 윗줄부터 시계방향으로 값 이동)을 정하고, layer별로 차근차근 구현하면 실수를 줄일 수 있다.
