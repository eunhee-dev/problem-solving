# 17135번. 캐슬 디펜스

## 문제
https://www.acmicpc.net/problem/17135

---

## Key Points

1. N, M이 15 이하이므로, 궁수 위치의 조합을 완전 탐색하여 죽일 수 있는 적의 최대값을 찾으면 된다.

    - 궁수 조합마다 사용할 `tmp_board`는 `board`를 복사하여 만든다.
        - 궁수 조합마다 `tmp_board`를 초기화 시켜주어야 하기 때문에, 2중 얕은 복사로 `board`를 복사한다.

    - 죽일 수 있는 적은 bfs로 찾으면 된다.
        - 아래서부터 위로 올라가며, 벽이 있는 것도 아니기 때문에 아래 방향은 고려하지 않아도 된다.
        - 최대 d 거리까지 죽일 수 있기 때문에, queue의 길이를 통해 이동 거리를 측정한다.
        - 가장 왼쪽의 적을 찾는 방법은, 탐색 순서를 왼쪽/위/오른쪽 순으로 정하면 가장 먼저 찾는 좌표가 가장 왼쪽에 위치한 적이 된다.
        - 적을 동시에 죽이는 방법은, `set()`을 통해 중복을 제거해주면 깔끔한 구현이 가능하다. 

    - 적이 한 행 내려오는 것은, `tmp_board.pop()`를 통해 쉽게 구현할 수 있다.
        - 새로운 적들이 추가되는 것이 아니고, bfs에서 n을 매번 새로 계산하기 때문에 `tmp_board`를 `pop()` 해주어도 된다.
        - 만약 새로운 적이 추가되는 문제였다면, `tmp_board`를 deque로 정의하고 `appendleft()`로 추가하면 좋다.

    - 어림잡아도 mC3에 대해 n*m bfs를 3번 돌리는거라 완전 탐색이 가능하다.
