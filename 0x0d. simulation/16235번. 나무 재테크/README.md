# 16235번. 나무 재테크

## 문제
https://www.acmicpc.net/problem/16235

---

## Key Points

1. 봄, 여름, 가을, 겨울에 할 일을 문제에서 제시한대로 구현해주면 되는 문제이다.

    - 봄에는 어린 나무 순으로 양분을 먹이고, 양분을 못먹은 나무는 죽어서 여름에 양분으로 변한다.

    - 가을에 나이가 5의 배수인 나무들은 번식을 하고, 겨울에는 양분이 추가된다.

2. 시간 복잡도를 맞추기 어려운 문제이다. 각 좌표마다 나무를 저장하는 자료구조를 잘 선택해야 한다.

    - K번 동안 N * N의 좌표에 대해 봄/여름/가을/겨울을 구현해야 한다.

    - 자료구조를 `list[list[deque[int]]]`로 구현하는 방법이 있다. [[풀이 - solve_naive.py](./solve_naive.py)]
        - 각 좌표마다 deque()으로 나무의 나이를 오름차순으로 저장한다.
        - 가을에 나무가 번식할 때, 나이 1짜리 나무를 deque.appendleft()로 왼쪽에 넣어줄 수 있다. (나이 순으로 정렬 유지 가능)
        - K * N^2 * 나무 개수 만큼의 시간 복잡도가 발생한다. 나무 개수 계산이 어려운데, 테스트 케이스에 따라 많이 커질 수 있어 시간 초과 가능성이 있다.
            - 실제로 Python3에서는 TLE가 났다.
            - 봄/가을에서 `tree_board[x][y]`는 deque()이지만 popleft()가 아닌 for문으로 나이를 꺼내는 것이 구현적으로 더 효율적인 것 같아 그렇게 했다.

    - 자료구조를 `defaultdict(int)`로 구현하는 방법이 있다. [[풀이 - solve.py](./solve.py)]
        - {나이: 나무 개수} 형태로 딕셔너리로 구성하는 방식이다.
        - 시간 복잡도는 K * N^2 * 나이 종류 수 * log(나이 종류 수)로, 나이의 종류가 나무 개수보다 훨씬 적기 때문에 더 효율적이다.
        - Python3, PyPy3 모두 통과하였다. (시간도 deque 방식보다 더 빨랐다.)
