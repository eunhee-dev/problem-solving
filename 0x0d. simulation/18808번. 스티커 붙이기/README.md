# 18808번. 스티커 붙이기

## 문제
https://www.acmicpc.net/problem/18808

---

## Key Points

1. 이 문제는 아래 두 단계로 나누어 생각하면 좋다.

   - (1) 스티커 붙여 보기: 노트북(`board`)의 왼쪽 위부터 차례대로 붙여보기 (`try_stick()`, `stick()`)

   - (2) 스티커를 못붙이면 회전시켜 보기 (`rotate()`) -> (1)로 돌아가기 (4번 반복)

2. 스티커 붙여보기(`try_stick()`)는 노트북의 왼쪽 위 좌표 (0, 0)부터 (n - r, m - c) 까지 `stick()`을 시도

   - `stick()`은 노트북에 스티커를 붙일 수 있는지 확인하는 부분, 실제로 붙이는 부분으로 나뉨

   - (r * c)를 (n * m)번 반복하는 작업

3. `rotate()`는 시계방향으로 90도씩 회전시킬 때 (x, y) -> (r - 1 - y, x) 로 바뀌는 규칙 활용하여 구현

4. 총 4번의 회전 * r(10) * c(10) * n(40) * c(40) * 스티커 개수(100) = 64000000 정도의 연산량으로 모든 경우의 수를 확인해도 통과 가능
