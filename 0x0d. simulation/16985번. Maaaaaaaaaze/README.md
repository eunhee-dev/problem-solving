# 16985번. Maaaaaaaaaze

## 문제
https://www.acmicpc.net/problem/16985

---

## Key Points

1. 문제를 처음 읽으면 어떻게 해야하나 막막할 수 있는데, 차분히 시간 복잡도부터 계산해보자.

    - 5 x 5 x 5라 판 5개를 각각 0, 90, 180, 270 회전하시킨 조합(4^5)을 가능한 모든 순서로 배치(5!)해도 122880 밖에 안된다.

    - 모든 조합에 대해 한 시작점에서 시작하는 bfs(5^3)를 돌리면 4^5 * 5! * 5^3 = 15360000 정도이다.

2. 모든 회전 + 순서 조합을 고려하기 때문에 시작점을 (0, 0, 0)으로 고정할 수 있다.

    - 다른 꼭지점에서 시작하는 경우는 다른 회전 + 순서 조합에서 다뤄진다.

    - 예를 들어 어떤 미로에서 (0, 0, 4)에서 시작하는 경우는 그 미로의 모든 판을 270도 돌린 미로에서 (0, 0, 0)에서 시작하는 것과 같다.

    - 이 개념이 조금 헷갈릴 수 있는데, 잘 생각해보면 꽤 쏠쏠한 최적화가 가능하다.
        - 8개 꼭지점을 모두 돌리는 사람은 없을 것 같고, 윗면 4개 꼭지점에 대해 돌려야 하나 고민되었을 것이다.
        - 결론적으로는, 하나의 시작점 (0, 0, 0)에 대해서만 고려해도 된다.

3. 우리는 이제 시간 복잡도가 넉넉하다는 것을 알았으니, 넉넉하게 구현해주면된다.

    - 5개의 판을 각각 0, 90, 180, 270 회전시켜서 저장하기 (`rotated_boards`)

    - 각 판의 회전 버전 뽑기 (`for selected_boards in product(*rotated_boards):`)
    
    - 뽑은 판들을 가능한 모든 순서로 배치하기 (`for maze in permutations(selected_boards):`) 

    - bfs를 통해 (0, 0 ,0)에서 (4, 4, 4)까지 이동 거리 구하기

    - 최소 이동 거리 구하기
        - 5 x 5 x 5에서 최소 이동 거리의 최솟값은 12이기 때문에, `min_dist == 12`인 경우 바로 return 해주는 소소한 최적화 가능
