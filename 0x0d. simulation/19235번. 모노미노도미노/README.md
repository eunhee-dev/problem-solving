# 19235번. 모노미노도미노

## 문제
https://www.acmicpc.net/problem/19235

---

## Key Points

0. 조건이 더 간단한 [20061번. 모노미노도미노 2](https://www.acmicpc.net/problem/20061)를 먼저 풀어보고 푸는 것이 좋을 수 있다.

    - 차이점: 20061번 문제는 완성된 행을 제거하고 그냥 한 칸 밀면 되지만, 본 문제는 제거 후 위에 있던 블록들에 대해 중력을 적용해야 한다.
        - 위의 행들의 블록들을 찾아서 중력을 적용해야 하므로, 각 블록이 어떤 블록인지 알고 있어야 함 => `block_id` 도입

1. 지문이 굉장히 길고 이해하기 어려운 문제이다. 차근차근 이해해보면 아래와 같이 요구사항을 정리할 수 있다.

    - 블록 놓기
        - 중력을 적용하여 떨어트리기

    - 완성된 행이 있는지 체크하여 제거하기
        - 제거 후, 중력을 적용하여 위에 있던 블록들 떨어트리기 (밑에 블록이 없거나 경계에 닿을 때 까지 떨어트려야 한다, 블록 놓기와 동일)
        - 완성된 행이 없을 때 까지 반복

    - 연한 행에 블록이 있다면 끝 행 제거하기

2. 파란색 보드는 초록색 보드와 (x, y) 좌표를 기준으로 대칭이다. 즉, 하나의 보드만 생각하고 구현하면 된다.

3. 블록 놓기 구현의 경우, 일단 블록을 (0, col)에 놓고 중력을 적용하는 방식으로 구현한다. (`add_block()`)

4. 완성된 행을 제거한 후, 중력을 적용할 때 고려해야할 요소가 많다.

    - 1x2 블록의 경우, 하나의 열이라도 밑에 블록이 있으면 그 위에 쌓여야 한다.

    - 즉, 1x2 블록에 대한 중력을 구현하기 위해서는, 보드에 있는 블록이 어떤 블록인지 구분되어야 한다.
        - `block_id`를 통해 구분

    - 완성된 행이 제거된 후의 중력을 적용할 때는 블록 놓기 코드를 활용한다.
        - 삭제된 행 바로 위 행부터 차례대로 (1) 어떤 종류의 블록인지 파악한 후 (블록 좌표: `(r, c)`), (2) 해당 블록을 지우고, (3) `add_block(board, t, r, c)`를 한다.
    

5. 완성된 행이 두 개가 있을 경우 동시에 제거되어야 한다.

    - 한 줄 씩 제거하는 방법: [[solve.py]](./solve.py)
        - `check_filled()` 함수에서 무조건 위 -> 아래 행 방향(`range(2, 6)`)으로 삭제해야 한다. 
        - [20061번. 모노미노도미노 2](../20061번.%20모노미노도미노%202/) 문제 풀이처럼, 아래 -> 위 방향(`range(5, 1, -1)`)으로 삭제하면 동시에 제거되는 구현이 안된다. (왜맞틀 할 수 있음...)
    
    - 동시에 제거하는 방법: [[solve_concurrent_removal.py]](solve_concurrent_removal.py)
        - `check_filled()` 함수에서 동시 제거 처리 및 score 계산
        - 방향을 고려하고 싶지 않으면, 이 방법이 더 편하다.
        - 제거된 행을 기억하기 위해서 코드가 약간 복잡해지는 단점이 있다. 하지만, 실수를 줄일 수 있다.
