# 16986번. 인싸들의 가위바위보

## 문제
https://www.acmicpc.net/problem/16986

---

## Key Points

1. 문제를 잘 이해해야 풀 수 있는 문제이다.

    - 지우, 경희, 민호 순으로 게임을 진행한다. => 순서 바꾸는 로직 구현 (`3 - p1 - p2`)

    - 무승부일 경우 민호 > 경희 > 지우 순으로 승리한다.

    - 지우는 서로 다른 손동작을 내야 승리한다. => 지우는 최대 N개의 손동작 안에 우승해야 함

    - 자신이 참여하는 경기에서 낼 손동작의 순서 처리 => 자료구조 결정

2. 지우는 최대 N(<= 9)개의 손동작 안에서 우승해야 하기 때문에 N!의 순열을 모두 탐색하면 지우의 우승이 가능한지 알 수 있다.

3. 자신이 참여하는 경기에서 낼 손동작의 순서의 자료구조를 `deque`으로 하는 방법, `list + index`를 활용하는 방법이 있다.

    - 왼쪽부터 손동작을 꺼내야 하기 때문에 `deque`을 쓰면 좋다. [[풀이 - solve.py](./solve.py)]
        - 코드의 가독성이 좋아지는 장점이 있지만, 조합마다 deque을 새로 만들어야 해서 효율성이 조금 떨어진다.
        - 데이터 분석 등 속도보다는 협업 환경에서는 이런 방식의 코딩이 좋은 것 같다.

    - 속도에 신경을 쓴다면 `list + index`를 쓰면 좋다. [[풀이 - solve_list_index.py](./solve_list_index.py)]
        - 속도는 확실히 빠르지만, `hand_idx`를 관리해줘야 하는 등 가독성은 deque에 비해 떨어진다.
        - PS와 같이 속도가 중요한 환경에서는ㄴ 이렇게 코딩하면 좋다.
