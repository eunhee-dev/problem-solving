# 10799번. 쇠막대기

## 문제
https://www.acmicpc.net/problem/10799

---

## Key Points

1. [9012번. 괄호](../9012/) 문제와 같이 괄호 종류가 하나이면 stack 대신 count하는 변수를 사용해도 된다.

2. ")" 괄호가 들어왔을 때, 직전 ch가 "(" 이면 레이저로, 아니면 쇠막대기가 끝난걸로 판단한다.

    - 레이저이면, 현재 쇠막대기 개수 (open_brackets) 만큼 추가된다.

    - 쇠막대기 끝이면, 끝난 쇠막대기 개수를 세어준다. (+1)

3. ")" 괄호가 들어왔을 때 직전 ch가 "(" 인지 체크하는 루틴에서 직전 ch를 미리 저장하고 있으면 좋다.

    - 보통 target_str[i - 1] == "(" 같은 방식을 떠올릴텐데, index off-by-one 에러의 여지가 있다.

    - 물론 input이 올바른 문자열만 들어오면 에러가 나진 않지만, prev = ch를 하면 애초에 그런 문제가 발생하지 않는다.

    - 같은 맥락으로 문자열 input을 신뢰하지 않으면, for문이 끝난 후 open_brackets != 0이면 ValueError를 띄우는 로직을 추가하는 것도 좋다.
