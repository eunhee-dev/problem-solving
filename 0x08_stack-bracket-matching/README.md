# 0x08. 스택의 활용 (괄호 매칭)

## 개요

- 스택은 괄호 매칭 문제에서 가장 대표적으로 활용된다.

- 여는 괄호를 스택에 push하고, 닫는 괄호가 나오면 스택의 top과 짝이 맞는지 확인한다.

- 괄호 종류가 하나면 스택 대신 count 변수로도 충분하다.

## 핵심 포인트

<details>
<summary>펼쳐 보기 (문제 풀이 스포일러 포함)</summary>

- **스택에 값도 저장**: 괄호의 값 문제처럼 괄호 외에 숫자도 스택에 넣으면 계산이 편하다.
    - 닫는 괄호가 나오면 앞의 숫자들을 모두 빼서 더하고, 해당 괄호값과 곱한다.
    - `(2)` → stack에 `["(", 2]` 형태로 저장

- **레이저와 막대기 구분**: 쇠막대기 문제에서 `()`는 레이저, 나머지는 막대기.
    - 직전 문자를 저장해두면 `target_str[i-1]` 접근 시 off-by-one 에러를 방지할 수 있다.

- **미완료 괄호 체크**: for문이 끝난 후 스택이 비어있는지(또는 괄호가 남아있는지) 확인한다.

</details>
