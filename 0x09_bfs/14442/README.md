# 14442번. 벽 부수고 이동하기 2

## 문제
https://www.acmicpc.net/problem/14442

---

## Key Points

1. [2206번. 벽 부수고 이동하기](../2206/) 문제의 [solve.py](../2206/solve.py) 풀이와 동일하다.

    - 기존 풀이와 약간 다르게 `dist` 배열을 사용하지 않고, `visited: list[list[bool]]` 배열을 사용하여 풀었음
        - bfs의 특징을 활용하여 queue의 길이를 이용하여 `distance`를 구함 (queue의 길이 만큼은 같은 distance임, 퍼져나가는 단계로 생각하면 이해하기 쉬움)
        - 아래 [solve_2d.py](./solve_2d.py) 풀이를 위해 `dist` 대신 `visited` 방식으로 구현하였음
        - 현재 distance를 매번 큐에 넣는 것보다 조금 더 효율적인 방법일 것으로 생각함

    - 시간복잡도 상 문제 없지만 Python3로 했을때 시간 초과 발생, PyPy3로 하면 통과 (백준의 이슈로 알고 있음, 실제로 반복 횟수가 많으면 PyPy3가 더 효율적임)

    - bytearray 등으로 해결이 가능해보이지만... 반복 횟수가 많을 때는 그냥 PyPy3를 쓰자 (아니면 C++...?)

2. [1600번. 말이 되고픈 원숭이](../1600/) 문제와 같이 2d-bfs로도 풀이가 가능하다. [solve_2d.py](./solve_2d.py)

    - 거리를 기록하는 3차원 배열 `dist` 대신, 2차원 `visited` 배열에 벽을 부순 최소 횟수를 기록하는 방식

    - 3d-bfs와 달리 broken_cnt에 대한 상태를 저장하고 있지 않기 때문에, 방문했는지만 체크해주면 되는게 아니라 `next_broken_cnt < visited[nx][ny]`와 같이 더 적게 벽을 부수고 갈 수 있는지도 체크해줘야 함
        - 기존 bfs처럼 방문했는지만 체크하면 큐에 들어가는 순서에 따라 더 최적의 경로를 찾지 못할 수 있음 (큐에 어떤 순서로 어떤 값이 들어가는지 생각해보자)
            - `경로 A`: `(0, 0)` -> `(1, 0)` (벽) -> `(1, 1)` (거리: 2, broken_cnt = 1)
            - `경로 B`: `(0, 0)` -> `(0, 1)` -> `(1, 0)` (거리: 2, broken_cnt = 0)
            - broken_cnt를 적게 사용한 `경로 B`가 더 최적일 가능성이 높은데, 방문했는지만 체크하면 경로 B에 대한 가능성을 고려하지 못함

        - `visited` 배열을 `inf`로 초기화했기 때문에 방문했는지도 동시에 체크 가능

    - 3차원의 경우 벽을 부순 횟수 별로 모두 방문하는 것이 허용되는데, 2차원은 벽을 부순 최소 횟수가 아니면 방문하지 않기 때문에 조금 더 효율적임
