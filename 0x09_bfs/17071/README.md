# 17071번. 숨바꼭질 5

## 문제
https://www.acmicpc.net/problem/17071

---

## Key Points

1. 시간마다 동생이 움직이기 때문에 일반적인 bfs 문제와 달리 타이밍까지 맞춰야 함

    - 시간에 관한 차원을 추가하는 아이디어 떠올리기

2. 가장 먼저 떠오른 naive 한 풀이 방식은 모든 시간에 대해 `dist[x][time]` 배열을 만드는 것이였음

    - MAX_POSITION = 500000 에 time이 대략 1000 정도라고 치면 시간/공간 초과 무조건 발생

3. `x-1`과 `x+1`로 이동할 수 있기 때문에, 2번 만에 제자리에 돌아올 수 있음

    - -1 하고 +1 하면 제자리에 있는 효과

    - 이로 인해, 0/1 (mod 2) 일 때 방문했던 좌표는 0/1 (mod 2)로 무조건 갈 수 있음

    - 다르게 말하면, 짝수/홀수 번째 시간에 방문했던 좌표는, 짝수/홀수 번째 시간에 무조건 방문할 수 있음

    - 짝수/홀수 번째 시간에 해당 좌표에 방문할 수 있는 최소 시간을 `dist`에 기록하면, 기록해둔 최소 시간 이후에 동생이 해당 좌표에 방문하더라도 타이밍을 맞출 수 있다.

4. 즉, 짝수/홀수 번째 시간에 해당 좌표에 방문할 수 있는 최소 시간을 bfs로 `dist` 배열에 기록해두고, 동생을 이동시켜보면서 `dist[curr_bro_loc][t % 2] <= t` 를 만족하는 최소 `t`를 찾으면 된다.
