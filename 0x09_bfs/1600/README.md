# 1600번. 말이 되고픈 원숭이

## 문제
https://www.acmicpc.net/problem/1600

---

## Key Points

1. [2206번. 벽 부수고 이동하기](../2206/) 문제와 비슷하게 3차원으로 풀이하면 편하다. [solve.py](./solve.py)

    - 벽 부수고 이동하기 문제에서는 벽을 부순 여부(횟수)를 세번째 차원에 기록했다면, 이 문제에서는 말 처럼 이동할 수 있는 횟수를 세번째 차원에 기록

    - 말처럼 이동할 수 있으면, 말처럼 이동해보고 `dist`에 기록하고 `queue`에 넣으면 된다.

2. 2d-bfs로 더 효율적인 풀이가 가능하다. [solve_2d.py](./solve_2d.py)

    - 거리를 기록하는 3차원 배열 `dist` 대신, 2차원 `visited` 배열에 말처럼 이동한 최소 횟수를 기록하는 방식

    - 기존 bfs나 [solve.py](./solve.py)의 3d-bfs 풀이에서는 해당 좌표에 방문했는지만 체크했다면, 이 방식에서는 해당 좌표에 최소로 말처럼 이동해서 왔는지를 체크해야 함
        - `horse_move_cnt` 상태를 따로 저장하지 않고 있기 때문에, 만약 해당 좌표를 이미 방문했는지만 체크한다면 가능한 경우를 모두 탐색하지 못할 수 있음
            - 참고로 3차원에서는 가능한 horse_move_cnt에 대해 다 방문해 봄 (모두 -1로 초기화 시켜놨기 때문에 방문한 적이 없는 것으로 인식)
        - 예를 들면, k=1인 경우에 (0, 0) -> (2, 1)를 말처럼 이동하여 거리 1로 왔는데, 마지막 도착점을 말처럼 이동하지 않으면 도착하지 못하는 경우; 결과: -1
          - 이 경우 말처럼 이동하기를 아껴서 (0, 0) -> (1, 0) -> (2, 0) -> (2, 1) 이렇게 거리 3으로 이동하고, 마지막에 말처럼 이동하면 도착점에 도착할 수 있음
        - 따라서, 기존처럼 방문한 점이라고 방문하지 않는게 아니라, 최소로 말처럼 이동해서 온 경우에는 **조금 더 느슨하게** 그 가능성을 열어두고 방문해보는 것
        - `if visited[nx][ny] > horse_move_cnt:` 부분이 이것에 해당
            - 아예 방문 안 한 경우도 방문해야 함; `visited`에 들어갈 수는 k+1보다 작을 것이기 때문에, `visited` 배열을 `inf (k+1)`로 초기화하면 최초 방문 보장

    - 3차원의 경우 말처럼 이동한 횟수 별로 모두 방문하는 것이 허용되는데, 2차원은 말처럼 이동한 최소 횟수가 아니면 방문하지 않기 때문에 조금 더 효율적임

3. 2d-bfs에서 `horse_move_cnt` 을 `visited[x][y]`에서 꺼내 쓰는 로직에 대해 한 번쯤 고민해보면 좋음

    - 기존까지 bfs에서는 이미 방문한 점이면 방문하지 않기 때문에 큐에 (x, y)가 딱 한 번 들어갔음

    - 근데 이 풀이의 경우 느슨하게 방문하기 때문에, (x, y)가 상황에 따라 큐에 여러 번 들어갈 수 있음 (`visited[x][y]`가 갱신될 수 있음)

    - 그 말은 (x, y)를 큐에 넣은 시점에서의 `horse_move_cnt`와 (x, y)를 큐에 꺼낸 시점에서 `horse_move_cnt`가 달라질 수도 있다는 것임
        - `visited[x][y]`가 그 사이에 갱신되었을 수 있음 
        - 다행히도, 말처럼 이동하는 것은 일반적인 이동보다 항상 앞서있고, 최소 횟수가 아니면 방문하지 않기 때문에 `visited[x][y]`가 그 사이에 갱신되지 않음

    - 만약 하루를 기다리면 순간 이동을 할 수 있는 등, 가만히 있을 수 있는 옵션이 생겨 순서가 꼬이는 경우에는 `horse_move_cnt`를 큐에 넣어줘어야 이전 상태가 추적 됨

    - 이 문제의 경우에는 순서가 꼬이지 않음; `horse_move_cnt`를 큐에 따로 안 넣고 `visited[x][y]`에서 꺼내 써도 무방함

    - 안전하게 큐에 넣는 방식도 있지만, 큐에 최소한으로 넣는 것이 최적화에 도움이 되므로 꺼내 쓰는 방식을 채택하였다.
