# 20304번. 비밀번호 제작

## 문제
https://www.acmicpc.net/problem/20304

---

## Key Points

1. 0 ~ n 까지 가능한 비밀번호 중 로그인 시도에 사용된 비밀번호 리스트 `p`의 값들과 안전 거리를 계산 했을 때, 가능한 가장 큰 안전 거리를 찾는 문제이다.

    - 여기서 안전 거리는 해밍 거리와 같은 의미이다.

2. n+1 길이의 `dist` 배열을 만들어, 0 <= x <= n 인 x에 대해, x와 p들의 값 중 최소 안전 거리를 `dist[x]`에 저장하고, `max(dist)`를 찾으면 된다.

    - x를 p의 값(안전 거리 = 0)들로 시작해서, bfs를 통해 안전 거리를 1씩 증가하여 `dist[nx]`에 안전 거리를 기록하면 된다.
        - 이미 `dist[nx]`가 기록되어 있으면, 다른 p_i가 더 작은 안전 거리를 적어둔 것임

    - nx 계산은 x에서 자릿수 k에 대해 xor 연산(`nx = x ^ (2**k)`)을 통해 각각 자리수를 차례로 1비트씩 바꿔주면 된다.
        - 예를 들어, 안전 거리 1인 값(`x`)들에서 1비트씩 바꿔주면 (=>`nx`) 안전 거리가 0 또는 2가 됨
        - 안전 거리 0인 값은 이미 `dist[nx]`가 기록되어 있어 무시 되므로, 안전 거리 2인 값들만 업데이트 해줄 수 있음

3. 솔직히 bfs workbook에 없었으면  bfs를 떠올리기 쉽지 않았을 것 같다. 각 비밀번호 후보들을 노드로 생각하는 참신한 발상을 해야하는 문제였다.
