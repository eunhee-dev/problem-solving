# 1697번. 숨바꼭질

## 문제
https://www.acmicpc.net/problem/1697

---

## Key Points

1. 1차원 bfs를 통해 동생까지 갈 수 있는 최단 시간을 찾는다.

2. 문제 조건에서 수빈이와 동생의 시작 조건만 있지, 이동 중에는 0 미만이나 100,000 이상으로 넘어갈 수 있다.

    - 따라서, 최대 200,000까지 체크해 줄 생각을 먼저 떠올리면 좋음

    - 하지만, 현재 2*x 위치로만 순간이동이 가능하기 때문에 100,000을 넘거나 0 미만으로 가는 것은 최단 시간일 수 없음
        - 100,000을 넘어가는 순간 -1로 내려와야 하는데, 그럴바에 -1을 먼저하고 2*x를 하는게 더 최적 선택
        - 0 미만으로 가려면 -1로만 갈 수 있기 때문에 굳이 내려갈 필요 없음

    - 그러므로, 이번 문제에서는 `MAX_POSITION = 100000` 로 설정해도 문제 없음

3. 수빈이와 동생이 같은 위치에서 시작할 수 있다. 문제 조건에 명확하게 명시되어 있지는 않지만 이럴때는 0이 나오게 해야한다.

4. 본 문제는 무조건 동생을 찾을 수 있기 때문에 while 문이 끝까지 돌 수 없다.

   - 따라서, 명시적인 의미를 담아 while 문 밑에 RuntimeError를 추가함
