# 13549번. 숨바꼭질 3

## 문제
https://www.acmicpc.net/problem/13549

## Key Points

1. 0-1 bfs 문제 (가중치가 0, 1인 bfs 문제)

    - 지금까지 풀었던 다차원 배열에서의 bfs 문제는 가중치가 모두 1임

    - 다익스트라로 풀 수 있지만, 가중치가 0, 1일 때는 0-1 bfs가 더 효율적인 방법 (O(n))

2. 가중치가 0인 순간이동(`2 * x`)의 경우 deq.appendleft()로 덱 앞에 추가하여 우선적으로 탐색될 수 있게 함

    - 우선적으로 탐색되지 않으면, 더 빠른 경우가 나중에 탐색되어 잘못된 결과를 낼 수 있음

    - `for nx in [2 * x, x - 1, x + 1]` 에서 `2 * x`의 순서는 중요하지 않지만, `x - 1`은 `x + 1` 보다 앞에 있어야 함
        - `2 * x`의 경우, 어처피 덱의 제일 앞에 넣어주므로 상관 없음
        - `x + 1`이 `x - 1` 보다 앞에 있으면, `2 * (x - 1)`의 거리가 생략된 채, `(x + 1) + 1`이 먼저 계산될 수 있음
        - 반례: 4, 6 => 답: 4 -> 3 -> 6 이지만, `x + 1`이 `x - 1` 보다 앞에 있으면: 4 -> 5 -> 6 이 먼저 계산

    - 위와 같은 순서 고려가 복잡하다면, 덱에서 뽑은 x에 대해 텔레포트를 끝까지 미리 다 시켜놓으면 효율은 조금 떨어져도 `x - 1`과 `x + 1`의 순서를 고려하지 않아도 됨

3. 가중치가 1인 경우에는 일반적인 bfs와 동일하게 deq.append()를 통해 순차적으로 탐색될 수 있게 함
