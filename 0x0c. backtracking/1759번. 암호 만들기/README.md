# 1759번. 암호 만들기

## 문제
https://www.acmicpc.net/problem/1759

---

## Key Points

1. [15655번. N과 M (6)](../15655번.%20N과%20M%20(6)/)의 응용 문제. [[풀이 - solve.py](./solve.py)]

    - `sequences`에 `path`를 넣을때, 자/모음 개수를 count하여 조건에 맞는 `path`만 넣는다.

2. `itertools`의 `combinations`를 사용하면 좀 더 간단하게 구현 가능하다. [[풀이 - solve_itertools.py](./solve_itertools.py)]

3. 위 풀이에서 `depth`를 `l`까지 들어간 다음에 자/모음 개수를 세는게 비효율적인 것 같다는 느낌이 들 수 있다. 그럴때는 가지치기를 해주면 좀 더 효율적인 코드를 짤 수 있다. [[풀이 - solve_with_pruning.py]](./solve_with_pruning.py)

    - `path`에 넣을 때 마다 자/모음 개수를 세고, 이미 자/모음 개수를 달성하지 못하는 경우를 미리 return 해버리면 좀 더 효율적이다.

    - 모음 개수 조건은 1개 이상이라 조금 번거로운데, 자음 개수 조건은 2개라 `cons_cnt` + `l - depth` (남은 슬롯)이 2보다 작은지만 가지치기 해줘도 꽤나 효율적이다.
        
        - `v_cnt + (l - depth) < 1` 은 `l == detph` 인 경우이라 밑에 있는 종료 조건과 중복된다.
        - 남은 구간의 자/모음 개수를 계산해두는 것도 가능하지만, 이 문제의 경우 입력이 작아 코드가 복잡해지는 것에 비해 큰 효과가 없을 것 같아 여기까지 하지는 않았다. (입력이 큰 경우에는 시도해볼만 하다)
            - 구현이 어렵지는 않다. 그냥 `chars`배열에서 모음이 더 이상 없는 index를 체크해두면 된다.

    - `c - start < l - depth`도 확인해주면 불필요한 재귀를 줄일 수 있다.
