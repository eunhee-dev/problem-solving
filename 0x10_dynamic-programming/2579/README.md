# 2579번. 계단 오르기

## 문제
https://www.acmicpc.net/problem/2579

---

## Key Points

1. 연속되는 3개의 계단을 밟으면 안된다는 상태 조건이 있기 때문에, 상태를 기록할 수 있는 2차원 dp 배열을 사용하면 된다. [[풀이 - solve.py](./solve.py)]

    - dp[i][j]: i번째 계단을 j번 연속 밟아서 올라왔을 때 얻을 수 있는 최대 점수

    - 두 계단 오를 경우: `dp[i][1] = max(dp[i - 2][1], dp[i - 2][2]) + stairs[i]`
        - 두 계단을 오르면 j는 1이 되고, i-2번째 계단에서 얻을 수 있는 점수의 최대 값 + 현재 계단 점수를 하면 된다.
    
    - 한 계단 오를 경우: `dp[i][2] = dp[i - 1][1] + stairs[i]`
        - 한 계단 오르면 j는 2가 되고, i-1번째 계단에서 1번 연속해서 밟았을 때 얻을 수 있는 점수 + 현재 계단 점수를 하면 된다.

2. 생각을 좀 더 해보면 1차원 DP로도 풀이 가능하다. [[풀이 - solve_1d_1.py](./solve_1d_1.py)]

    - dp[i]: i번째 계단을 밟았을 때 얻을 수 있는 최대 점수

    - i번째 계단을 밟기 위해서는 두가지 경우의 수가 가능하다.
        - i-2번째 계단에서 2칸 오르는 경우(`dp[i - 2] + stairs[i]`): i-1번째 계단을 밟지 않아 연속 3칸 규칙에 위배되지 않음
        - i-1번째 계단에서 1칸 오르는 경우 => i-3번째 계단에서 2칸 올라야 연속 3칸 규칙에 위배되지 않음 (`dp[i - 3] + stairs[i - 1] + stairs[i]`)
    
    - 따라서, 최종 점화식은 `dp[i] = max(dp[i - 2], dp[i - 3] + stairs[i - 1]) + stairs[i]`이다.

3. 문제를 반대로 생각해서, 밟지 않을(skip할) 계단의 최소값을 구하는 방식도 가능하다. [[풀이 - solve_1d_2.py](./solve_1d_2.py)]

    - 발상의 전환이라 약간 어려울 수 있어 차근차근 이해해봐야 한다.

    - dp[i]: i번째 계단을 밟지 않았을 때, 밟지 않은 계단의 최소값

    - i번째 계단을 밟지 않기 때문에, i-1번째 계단을 반드시 밟아야 함 (최대 2칸 밖에 점프할 수 없으므로, i번째 계단과 i-1번째 계단을 동시에 skip할 수 없기 때문.)

    - i-1번째 계단을 밟는 방법은 아래와 같다.
        - i-3 => i-1: i-2번째를 skip (`dp[i - 2]`)
        - i-4 => i-2 -> i-1: i-3번째를 skip (`dp[i - 3]`)

    - 즉, i번째 계단을 밟지 않기 위한 점화식은 `dp[i] = min(dp[i - 2], dp[i - 3]) + stairs[i]`

    - n번째 계단을 밟았을 때 최대 점수 합은 `sum(stairs) - min(dp[i - 1], dp[i - 2])`임
        - i-1번째를 skip하거나 (i-2 => i) i-2번째를 skip한 경우(i-3 => i-1 => i) 중 최소값을 전체 계단 합에서 빼면 최대값임
