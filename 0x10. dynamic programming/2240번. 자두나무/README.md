# 2240번. 자두나무

## 문제
https://www.acmicpc.net/problem/2240

---

## Key Points

1. 시간, 움직인 횟수, 현재 위치 3가지 상태를 담은 3차원 dp를 활용하여 풀이 가능하다.

    - dp[i][j][k]: i 시간에 j번 움직였을 때 k번 나무 위치에 있는 경우 얻을 수 있는 자두의 최대 개수

    - `dp[i][j][plum_loc] = max(dp[i - 1][j][plum_loc], dp[i - 1][j - 1][other_loc]) + 1`
        - `plum_loc`: 현재 자두가 떨어지는 나무 위치, `other_loc`: `plum_loc이` 아닌 나무
        - j == 0 인 경우에는 `dp[i][j][plum_loc] = dp[i - 1][j][plum_loc]` + 1 이다.
    
    - `dp[i][j][other_loc] = dp[i - 1][j][other_loc]`
        - 이 경우에도 위에처럼 max를 비교해야 되지만 (dp 테이블을 채우는 관점에서는 그렇게 해야함),
        - 의미를 생각해보면, `plum_loc`(자두가 떨어지는 나무)에서 굳이 움직여서 자두를 피하는 행위는 최적해가 될 수 없기 때문에 가만히 있는 경우만 고려하였다.
    

2. 현재 문제에서는 자두의 시작 위치가 고정되어 있고, 나무가 2개밖에 없기 때문에 홀짝성을 이용하여 k가 없는 2차원 dp 배열로도 구현 가능하다.

    - 1번 나무에서 시작하기 때문에 j가 짝수이면 1번 나무에 있고, 홀수이면 2번 나무에 있기 때문에 현재 위치를 기록하지 않고 구할 수 있다.

    - 반대로 말하면 시작 위치가 고정되지 않거나, 나무가 3개 이상인 경우 3차원 dp로 풀어야하기 때문에 좀 더 일반적인 3차원 dp로 풀이하였다.
