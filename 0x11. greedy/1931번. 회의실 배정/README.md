# 1931번. 회의실 배정

## 문제
https://www.acmicpc.net/problem/1931

---

## Key Points

1. 회의가 **끝나는 시간이 가장 빠른** 순으로 정렬한 후 그리디하게 고르면 된다.

    - 회의에 들어갈 수 있는 시점에서 가장 빨리 끝나는 회의를 배정한다. (하나라도 회의 개수를 더 늘리기 위해, 귀류법으로 증명 가능)

2. 회의 시작시간과 끝나는 시간이 같을 수 있다. 그런 케이스를 놓치지 않기 위해 [끝나는 시간 -> **시작 시간**] 순으로 정렬하면 구현이 수월해진다.
