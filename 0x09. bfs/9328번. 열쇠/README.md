# 9328번. 열쇠

## 문제
https://www.acmicpc.net/problem/9328

---

## Key Points

1. 기본적인 아이디어는 [11967번. 불켜기](../11967번.%20불켜기/) 문제와 유사하다고 볼 수 있다.

    - 키를 발견하면, 방문을 시도했지만 해당 키가 없어 열지 못한 문으로 순간 이동 (큐에 넣음)

    - 방문을 시도했지만 키가 없어 열지 못한 문을 기록해두어야함 (defaultdict 사용, `locked_doors`)

2. 먼저 테두리 좌표 중에서 가능한 출발점들을 뽑고, 출발점들에 대해 bfs를 돌리면 된다. [풀이 - solve_naive.py](./solve_naive.py)

    - 출발점에 `$`나 key, `locked_doors`가 있는 경우를 잘 고려하여 구현해야 한다.

3. 조금 더 생각해보면, 테두리 좌표에 벽이나 `locked_doors`를 제외하고 자유롭게 드나들 수 있기 때문에, board를 `.` 으로 둘러싸면 bfs를 한 번만 돌려도 된다. [풀이 - solve.py](./solve.py)

    - input으로 들어온 board를 `.`으로 둘러싸면 (0, 0)에서 시작하는 bfs를 한 번만 수행해도 된다.

    - [solve_naive.py](./solve_naive.py)와 같이 테두리 좌표를 뽑는다거나, 출발점에 뭐가 있는지와 같은 고민을 전혀 안해 된다.

    - 좋은 아이디어로 인해 구현이 매우 간단해진다.
