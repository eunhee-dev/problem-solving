# 2146번. 다리 만들기

## 문제
https://www.acmicpc.net/problem/2146

---

## Key Points

1. 섬을 구분하기 위해 먼저 bfs를 통해 섬 구분 작업 (라벨링) 수행

2. 직관적인 풀이는 [solve.py](./solve.py)와 같은 방식임

    - 섬에 해당하는 좌표에 대해 전부 bfs를 수행하여 다른 섬까지의 최소 경로를 찾음

    - 섬에 테두리에 대해서만 진행하면 약간의 최적화 가능 (테두리가 더 최소일거니까)

    - 이 방법은 O(n^4)의 시간 복잡도를 가짐 (섬 n^2개 대해 각각 bfs O(n^2) 수행)

3. 좀 더 효율적인 방법은 [solve2.py](./solve2.py)와 같은 방식임

    - [4179번. 불!](../4179번.%20불!/), [7576번. 토마토](../7576번.%20토마토/) 문제에서의 멀티소스 bfs 개념 적용

    - 각 섬의 테두리에 해당하는 좌표를 전부 큐에 넣고 bfs 수행

    - 바다에 해당하는 좌표를 해당 섬으로 표시 (땅따먹기, 섬 영역 확장 느낌)

    - 만약 다른 섬에 해당하는 좌표에 도달하면 해당 좌표에 적혀있는 거리와 현재까지 거기를 더하여 다리 길이 계산, 최소의 다리 길이를 찾음
        - 다른 섬의 시작점으로부터의 거리가 해당 좌표에 적혀있기 때문에 중간에서 만나는 효과를 가짐

    - 이 방법은 O(n^2)의 시간복잡도를 가짐 (bfs를 한 번만 수행, queue에 최대 n^2개가 들어감)
