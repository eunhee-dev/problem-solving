# 10989번. 수 정렬하기 3

## 문제
https://www.acmicpc.net/problem/10989

---

## Key Points

1. N이 엄청 큰데, 수가 10,000보다 작다는 조건이 있으므로 계수 정렬을 사용하면 좋다.

    - 숫자를 카운팅 하면, `counts` 배열을 10001개로 유지할 수 있어 메모리 초과가 발생하지 않는다.

    - 기존의 `main()`에서만 입출력을 받는 코드 규칙은 이 문제에서는 사용할 수 없었다. (입력을 가지고 있으면 메모리 초과 발생, `counts`에 반영하고 버려야 함)
