# 7795번. 먹을 것인가 먹힐 것인가

## 문제
https://www.acmicpc.net/problem/7795

---

## Key Points

1. 정렬을 사용하지 않고, 이중 for문으로 구현하면 O(n x m)으로 시간 초과가 발생할 수 있다.

2. a와 b를 정렬하고, a의 값을 하나씩 꺼내어 b에 있는 값과 비교해본다.

    - a와 b가 정렬되어 있기 때문에, `a[i]`보다 작은 b의 원소들은 당연히 `a[i + 1]`보다도 작다.
        - a가 정렬되어 `a[i + 1] >= a[i]` 이므로, 만약 `a[i] > b[j]` 라면 `a[i + 1] > b[j]` 도 항상 성립한다.
        
    - 즉, `a[i]`에 대한 탐색을 마쳤을 때 b의 포인터가 `b_idx`에 있다면, `a[i + 1]`에 대한 탐색은 b의 처음부터가 아니라 `b_idx`부터 이어서 하면 된다.
        - `b_idx` 이전의 원소들은 이미 `a[i + 1]`과의 쌍으로 확정되었기 때문이다. (`a[i + 1] >= a[i] > b[b_idx - 1]`)

    - 이 방식으로, 시간복잡도를 O(n x m)에서 O(n + m)으로 줄일 수 있다.
    
    - 이런 방식의 알고리즘을 [투 포인터](https://blog.encrypted.gg/1004)라고 한다.
