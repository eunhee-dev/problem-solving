# 0x0b. 재귀

## 개요

- 재귀를 무조건 사용해야 하는 경우는 없다. 이론적으로 대부분의 재귀는 반복문으로도 구현이 가능하다.

- 오히려 재귀의 깊이가 깊어지면 스택에 부담을 줄 수 있다. 비슷한 가독성의 반복문으로 구현 가능하다면, 반복문으로 구현하는 것이 안전하다.

- 하지만, 재귀를 사용해서 코드가 더 직관적이거나, 구현 난이도를 낮춰주는 경우가 있다. (DFS, 백트래킹, 분할 정복 등)

## 핵심 포인트

<details>
<summary>펼쳐 보기 (문제 풀이 스포일러 포함)</summary>

- **분할 정복**: 문제를 더 작은 부분 문제로 나누어 해결한다.
    - 거듭제곱: `a^(2b) = a^b * a^b` 활용 (Python은 `pow(a, b, c)` 지원)
    - Z 탐색: 4등분하여 해당 사분면의 offset 계산

- **Python 재귀 제한**: 백준에서 `sys.getrecursionlimit()`은 기본 1000이므로, 재귀 깊이가 깊어질 수 있는 문제는 `sys.setrecursionlimit()`으로 조정 필요.

- **반복문 변환**: 재귀로 구현한 것은 대부분 반복문으로 변환 가능. `pow()` 같은 빌트인 함수도 내부적으로 반복문으로 구현됨.

</details>
