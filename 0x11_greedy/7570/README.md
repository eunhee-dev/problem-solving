# 7570번. 줄 세우기

## 문제
https://www.acmicpc.net/problem/7570

---

## Key Points

1. 맨앞/뒤 삽입 연산은 움직이지 않는 원소들의 상대적 순서를 보존한다. 

    - 따라서 움직이지 않아도 되는 원소들의 조건은 아래와 같다.
        - 값이 연속이고 (예: 2,3,4),
        - 배열에서 증가하는 순서로 나타나야 함

    - 예: 5 2 4 1 3 에서 2 3은 움직이지 않아도 됨 / 5 4 1은 올바른 위치에 있지 않으므로 움직여야 함

    - 예: 5 2 3 1 4 에서 2 3 4는 움직이지 않아도 됨 / 5 1은 움직여야 함

    - 예: 5 4 3 2 1 에서 1은 움직이지 않아도 됨 / 5 4 3 2는 움직여야 함

2. 즉, "가장 긴 연속으로 증가하는 부분 수열"을 찾으면 되는 문제이다.

    - 연속 증가 수열에 포함되지 않는 어린이는 앞이나 뒤로 보내면 됨
        - 연속 증가 수열을 `S = {a, a+1, ..., a+L-1}`이라 하면,
        - `1, 2, ..., a-1`을 큰 값부터 순서대로 맨 앞에 삽입
        - `a+L, a+L+1, ..., N`을 작은 값부터 순서대로 맨 뒤에 삽입

    - 즉, 최소 이동 횟수는 `N - len(가장 긴 연속으로 증가하는 부분 수열)`이 된다.

    - 가장 긴 연속으로 증가하는 부분 수열은 dp를 활용하여 구할 수 있다.
        - dp[v] = 값 v로 끝나는 연속 증가 부분 수열의 길이
        - 배열을 앞에서부터 순회하면서 `dp[v] = dp[v-1] + 1`
        - `dp[v-1]`이 이미 갱신되어 있다는 건 값 `v-1`이 `v`보다 앞에 있다는 뜻이므로, 자연스럽게 "앞에 있으면서 값이 연속"인 조건을 만족함

        ```
        [예시]

        입력: 5 2 4 1 3

        v=5: dp[5] = dp[4] + 1 = 1  (4가 아직 안 나옴)
        v=2: dp[2] = dp[1] + 1 = 1  (1이 아직 안 나옴)
        v=4: dp[4] = dp[3] + 1 = 1  (3이 아직 안 나옴)
        v=1: dp[1] = dp[0] + 1 = 1
        v=3: dp[3] = dp[2] + 1 = 2  (2가 이미 나왔고 dp[2]=1)

        max(dp) = 2 → 답 = 5 - 2 = 3
        ```
