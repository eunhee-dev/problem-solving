# 1700번. 멀티탭 스케줄링

## 문제
https://www.acmicpc.net/problem/1700

---

## Key Points

1. n과 k 제한이 100으로 작기 때문에 넉넉하게 풀이할 수 있다.

2. 꽂아야 할 기기를 순서대로 멀티탭에 안 꽂혀 있으면 꽂고, 꽂혀 있으면 넘어간다.

    - 멀티탭이 꽉 차 있으면 꽂혀 있는 플러그 중 가장 비효율 적인 플러그를 뽑는다. (`앞으로 꽂을 일 없는 기기` -> (없으면) `가장 나중에 꽂아야 할 기기`)
        - 이 방식은 `Optimal Page Replacement` 알고리즘과 동일하며, 플러그 교체 횟수를 수학적으로 최소화함을 보장한다.

    - 뽑아야 할 플러그를 찾는 작업에 $O(N \cdot K)$이 소요된다. 전체 스케줄을 도는 외부 루프$(K)$까지 포함하여, 전체 시간 복잡도는 $O(N \cdot K^2)$가 된다.
