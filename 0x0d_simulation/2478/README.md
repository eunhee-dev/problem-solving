# 2478번. 자물쇠

## 문제
https://www.acmicpc.net/problem/2478

---

## Key Points

1. 주어진 자물쇠를 역순으로 (오른쪽으로 밀기, k2) -> (구간 뒤집기, p ~ q) -> (오른쪽으로 밀기, k1)해서 [1, ..., n]이 되는 값을 찾을 수 있지만, N이 500이라 시간 초과가 발생한다.

    - 오른쪽으로 밀기 경우의 수: n x 2번 + 구간 뒤집기: nC2로 총 O(n^4)의 시간 복잡도를 가진다.

    - 구간 뒤집기 후 오른쪽으로 밀기(k1)를 굳이 n번 탐색하지 않아도, O(n^3)의 시간 복잡도를 가진다.

    - O(n^2) 정도만 되어도 안정적으로 통과 가능하다.

2. 구간 뒤집기의 (p, q)를 구하는 방법에 대해 고민해 봐야한다.

    - 첫 번째 오른쪽 밀기(k1)를 해도 자물쇠는 1씩 증가하는 오름차순이 cyclic하게 유지된다.

    - 여기서 구간 뒤집기를 하면, 1씩 증가하는 오름차순이 깨지는 지점이 생긴다.
        - 뒤집힌 구간은 1씩 감소하는 내림차순이 된다.
        - (예외) 전체 구간을 뒤집으면 전부 1씩 감소하는 내림차순이 된다.

    - 즉, 두 번째 오른쪽 밀기 전에 p: 1씩 증가하는 오름차순이 깨지는 지점 (내림차순의 시작) / q: 1씩 감소하는 내림차순이 깨지는 지점으로 뒤집힌 구간 (p, q)를 구할 수 있다.
        - (예외) 전체 구간을 뒤집은 경우는 내림차순이 깨지는 지점이 없다. 이럴 때는 p: 0 / q: n-1로 잡으면 된다.
        - 위 과정은 O(N)의 시간 복잡도를 가진다.
        - 두 번째 밀기가 없었으면 여기서 쉽게 끝나지만, 두 번째 밀기를 통해 (p, q)의 값이 바뀔 수 있어 문제가 좀 더 복잡하다. (p < q 를 유지해야 하기 때문)

    - 적당한 k2를 뽑고, k2만큼 오른쪽으로 민 다음 조건에 맞는 뒤집힌 구간 (p, q)를 구하면 된다.
        - k1은 굳이 n번 탐색할 필요 없이 제일 앞 숫자만을 보고 구할 수 있다. (제일 앞 숫자가 자기 자리를 찾아가기 위해서는 그 숫자만큼 밀면 된다. 이 값이 k1이다.)
        - 따라서 적당한 k2는 brute force 해봐도, 총 시간복잡도는 O(N^2)으로 넉넉하다.

3. 2번의 아이디어를 활용하여 다음과 같이 구현할 수 있다.

    - index 계산의 편의성을 위해 1-based인 현재 자물쇠를 0-based로 바꾼다.

    - 구현을 쉽게 하기 위해, k2를 brute force 하여 1 ~ n-1 까지 오른쪽으로 밀어본다. O(n)

        - 1씩 증가하는 오름차순이 깨지는 구간과, 1씩 감소하는 내림차순이 깨지는 구간을 찾는다. O(n)
            - 각각을 p, q로 설정한다.
            - 첫 번째 구간 뒤집기 후에도 1씩 증가하는 오름차순이 유지되기 때문에, 앞에서부터 봤을 때 오름차순이 깨지는 지점 p가 무조건 먼저 나온다.
            - 1씩 감소하는 내림차순이 깨지는 경우가 없으면 (p, q) = (0, n-1)로 설정한다.

        - (p, q) 구간을 뒤집는다.

        - 현재 좌물쇠의 제일 앞 숫자를 확인하여, (자기 자리를 찾아가도록) 그 숫자(k1)만큼 오른쪽으로 민다.
            - 제일 앞 숫자가 0이면 버린다. (1 <= k1 < n 이기 때문에 k1이 0인 경우는 고려하지 않는다.)

        - 완성된 좌물쇠가 0 ~ n - 1의 숫자인지 확인한다. 맞으면 방금 구한 k1, (p, q), k2가 정답이 된다.

4. 위 풀이를 잘 생각해보면 O(n)로 풀이할 수도 있겠다는 생각이 든다.

    - 잠겨진 자물쇠에서 오름차순이 깨지는 지점 / 내림차순이 깨지는 지점인 (p, q) 구간을 찾고, 적당한 k1, k2를 계산하는 방식이 가능하다.
        - 적당한 k1, k2란 p < q 를 만족하며, 1 <= k1, k2 < n 을 만족하는 값

    - 적당한 k1, k2를 계산하는 것이 생각보다 복잡하다. 그리고, k1, k2 값에 따라 (p, q)도 조정해주어야 한다.
        - [solve.py](./solve.py) 풀이는 k2를 brute force 함으로써 이러한 복잡성을 제거하였다.
        - 실제로 이 풀이는 O(n^2) 보다는 더 빠를텐데, 조건에 맞는 k1, k2를 찾으면 바로 종료되기 때문이다. (적당한 k1, k2를 자동으로 찾아주는 느낌, 적당한 k1, k2는 꽤 많다.)

    - O(n) 풀이는 [solve_n.py](./solve_n.py)와 같이 풀 수 있다.
        - 두 번째 밀기로 인해, 앞에서 부터 봤을 때 오름차순이 깨지는 지점이 먼저 나오지 않을 수 있다. (p > q)
            - 따라서, 주어진 자물쇠에서 오름차순 구간과 내림차순 구간을 먼저 파악하는 작업이 필요하다. (`trends`)
            - p: `+` -> `-` 변곡점 / q: `-` -> `+` 변곡점
        - k2를 1씩 증가시키면서 p < q 이며, k1이 0이 되지 않는 값을 구한다.
            - k2도 0이 되면 안되기 때문에 무조건 한 번은 밀어야 한다. 따라서, 전체 구간을 뒤집은 경우 p, q를 각각 n-1, n-2로 설정하면 좋다. (오른쪽으로 한 번 밀면 0, n-1이 되기 때문)
