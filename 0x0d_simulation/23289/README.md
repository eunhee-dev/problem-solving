## Key Points

1. 다음과 같이 나누어 생각하면 좋다. 구현이 복잡하기 때문에, 각 기능을 독립적으로 테스트해보는 것이 좋다.

    - 온풍기 가동하기: bfs 활용

    - 온도 조절하기: `diff_board`를 두어 동시에 계산되도록 하기

    - 바깥쪽 칸의 온도 1 감소하기

    - 조사하기

2. 벽에 막혀 이동할 수 없는지 확인하기 위해 이중 set을 사용하면 좋다.

    - 벽을 {(x1, y1), (x2, y2)}와 같이 set으로 저장하면, 두 좌표의 순서가 바뀌어도 확인 가능하다. [(x1, y1)과 (x2, y2)는 인접한 좌표] 

    - 벽들의 집합을 {{(x1, y1), (x2, y2)} ...} 형식으로 저장하면, 인접한 두 좌표 {(x, y), (nx, ny)}가 벽들의 집합에 들어있는지 O(1)으로 확인할 수 있다. 

    - python에서 이중 set을 지원하지 않기 때문에, `set[frozenset[Coord]]` 방식으로 구현한다.
        - set은 mutable하기 때문에 hashable하지 않다. 하지만, set 안에 들어가는 값은 hashable 해야 한다.
        - frozenset은 immutable하기 때문에 hashable 하다.
        - 벽에 해당하는 {(x1, y1), (x2, y2)}는 변하지 않는 고정 값이다.
        - 따라서 `set[frozenset]` 형태로 이중 set을 구현할 수 있다.

3. 최적화 요소

    - 온풍기를 가동한 결과는 동일하기 때문에, 한 번만 계산하여 `heat_board`에 저장해두면 효율적이다.

    - 온도 조절하기에서 인접한 좌표를 계산할 때, 오른쪽과 아래 방향만 확인해주면 더 효율적이다.
        - (x, y)와 (x, y+1)의 온도차를 계산하고, 나중에 (x, y+1)에서 (x, y)를 확인하면 중복 계산이기 때문
        - 소수점 이하를 버리는 구현은 음수를 고려하여 `//` 대신 `int()`를 활용하면 실수를 예방할 수 있다.
