# 16637번. 괄호 추가하기

## 문제
https://www.acmicpc.net/problem/16637

---

## Key Points

1. 괄호를 어떻게 쳐야할지 막막할 수 있는데, 괄호에는 연산자가 하나만 들어갈 수 있다는 조건이 있으므로 연산자를 고른다고 생각하면 좋다.

    - 연산자를 고르면, 그 연산자의 앞 뒤 숫자를 괄호치는 것이다.

    - 괄호 문제 => 연산자 조합 문제로 바뀜

    - N <= 19 이므로 연산자는 최대 9개 있을 수 있어 완전 탐색이 가능하다. (대충 잡아도 2^9 미만)

2. 백트래킹을 사용하면 연산자를 골랐을 때와 안 골랐을 때의 모든 경우를 탐색하기 좋다.

    - 괄호 안치고 다음 단계로 보내기 (`depth + 1`)

    - 괄호 치고 다음 단계로 보내기 (괄호에 들어갈 다음 연산자를 미리 계산하기 때문에 `depth + 2`)

3. 코드를 보면 피보나치의 성질을 갖고 있음을 알 수 있다.

    - T(m): 남은 연산자의 개수가 m개일 때 backtrack 함수가 만드는 총 경우의 수

    - T(m) = T(m - 1) + T(m - 2)

    - 점화식이 세워졌으니 dp로도 풀 수 있다.
