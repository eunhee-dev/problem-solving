# 17070번. 파이프 옮기기 1

## 문제
https://www.acmicpc.net/problem/17070

---

## Key Points

1. N이 16 이하이므로, 완전 탐색(시뮬레이션)을 하면 아슬아슬하게 통과할 것 같다.

2. 하지만, 이 문제는 시뮬레이션 + 완전 탐색 기준 python으로 구현하기 까다로운 문제이다.

    - Python3로는 완전 탐색 풀이가 불가능한 듯 하다. (반복이 많은 문제는, JIT 컴파일이 적용된 PyPy3가 보통 유리하다.)

    - PyPy3의 경우에도 도착점이 벽인 경우를 예외처리 해주지 않으면, 시간 초과가 발생한다. [[풀이 - solve.py](./solve.py)]

    - 이는 python에서의 재귀 호출 비용 + 컴파일러/인터프리터 레벨에서의 구현 차이 등 굉장히 복잡 미묘한 부분이다.

    - 예를들어, dfs 구현을 재귀가 아닌 스택으로 구현하면 도착점이 벽인 경우 예외처리 없이 통과한다. [[풀이 - solve_stack_bfs.py](./solve_stack_dfs.py)]
        - python에서는 재귀보다 스택으로 dfs를 구현하는게 더 효율적인 것 같다.

    - dfs를 재귀로 구현해도, 파이프 종류별로 if/elif/else를 나누면 예외처리 없이 통과한다. [[풀이 - solve_verbose.py](./solve_verbose.py)]
        - 별로 큰 차이는 없어보이지만, 컴파일러/인터프리터 레벨에서의 미묘한 구현 차이가 많이 반복되어서 성공/실패의 차이를 만든 것 같다.
        - 중복되는 코드 조각이 많아 지저분해지는 단점이 있다.
        - 코드 중복을 없애기 위해 함수로 바꾸면, 함수 호출비용 때문에 시간초과가 발생한다.

    - 이처럼 시뮬레이션으로 완전 탐색할 경우, python으로 구현하기 굉장히 복잡하며 미묘한 문제이다. (예외처리, 최적화 등 많이 신경써주어야 함, 그래도 안될 수 있음)

    - 지금 문제의 경우, 무작정 탐색해보는 것이 아니라 해봤던 것을 기록(memoization)함으로써 반복을 줄이면 여유롭게 통과한다. [[풀이 - solve_memo.py](./solve_memo.py)]
        - Python3 기준 40ms 정도로 통과한다.
        - 다만, 이 특성을 알아채면 아래와 같이 DP 풀이를 자연스럽게 떠올리게 될 것이다.

3. 보통 이런 경우에는 Dynamic Programming (DP)를 사용하면 간단하게 해결되는 경우가 꽤 있다. [[풀이 - solve_dp.py](./solve_dp.py)]

    - dp[i][x][y]: i 종류의 파이프로 (x, y) 좌표에 도달할 수 있는 방법의 수

    - 완전 탐색에서는 해봤던 것을 기록하지 않아 불필요한 반복이 많은데, dp를 사용하면 그러한 반복을 효과적으로 줄일 수 있다.

    - C/C++을 사용하면 완전 탐색으로 쉽게 풀리는 부분이, python의 경우 한 단계 더 나아가서 memoization이나 dp를 사용해야하는 경우가 종종 있다.

    - dp가 더 효율적이기도 하고 구현이 간단하므로, python 유저라면 DP로 풀 수 있는 문제는 DP로 푸는 것이 정신 건강에 좋다.

    - DP는 강의 순서 상 다음 단원이니, DP를 아직 학습하지 않았다면 학습 후 다시 이 문제로 돌아와서 풀어보면 좋다.
