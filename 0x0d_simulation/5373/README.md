# 5373번. 큐빙

## 문제
https://www.acmicpc.net/problem/5373

---

## Key Points

1. 각 면을 기준으로 돌렸을 때, (1) 기준 면과 (2) 기준 면과 접하는 면이 어떻게 바뀌는지 구현하면 되는 문제이다.

    - (1) 기준 면은 그냥 2차원 배열을 90도로 돌리는 것과 같다.

    - (2) 기준 면과 접하는 면들은 하나의 행 또는 열이 바뀐다. 규칙을 잘 파악하여 구현해줘야 한다.

2. 풀이 아이디어 자체는 어렵지 않지만, 인덱스 계산이 꽤나 어려운 문제이다. (집에 큐브가 있거나, 공간지각력 요구)

    - 각 면의 인덱스를 자신의 기준에 따라 명확하게 지정해줘야 한다.

        - 나의 경우, UP -> FRONT -> DOWN -> BACK을 기준으로 인덱스를 부여하였다. (DOWN의 0번 행이 FRONT의 2번 행과 접하도록, BACK의 2번 행이 UP의 0번 행과 접하도록)

        - LEFT와 RIGHT는 UP과 접하는 행을 0번 행으로 지정했다.

    - 자신이 지정한 인덱스를 기준으로, 각 면을 기준으로 돌렸을 때 접하는 면들이 어떻게 배치되는지 코드로 작성한다.

3. (미세 Tip) 시계 방향만 구현하고, 반시계 방향은 시계방향 * 3번 해주면 구현을 조금 적게 해도 된다.
