# 15684번. 사다리 조작

## 문제
https://www.acmicpc.net/problem/15684

---

## Key Points

1. 이 문제는 다음 단계로 나누어 생각하면 좋다.

    - 추가할 사다리 (조합) 뽑기

    - 뽑은 사다리를 놓을 수 있는지 확인하기 (인접하지 않은지)

    - 현재 사다리로 i번 세로선의 결과가 i번이 나오는지 체크하기 (`check()`)

2. 백트래킹을 통해 사다리를 뽑을 수 있다.

    - 여기서 방법이 나뉜다.
        1. 최소 사다리 개수가 몇개인지 모르기 때문에 depth가 깊어질 때마다 `check()` 해주는 방식 ([[풀이 - solve_naive_backtrack.py](./solve_naive_backtrack.py)])
        
        2. 최소 사다리 개수를 0 ~ 3개로 설정하여 차례대로 백트래킹을 돌려보는 방식 ([[풀이 - solve.py](./solve.py)]) 

    - 언뜻보면 1번은 이전 깊이의 결과를 재사용하고, 2번은 깊이가 달라지면 다시 새로 계산하기 때문에 1번이 효율적으로 보일지 모르지만, `check()` 함수의 호출이 비싸기 때문에 2번이 훨씬 효율적이다.
        - 1번은 `check()` 함수를 매 깊이마다 호출한다.
        - 반면에, 2번은 `check()`를 깊이가 `count`가 될때만 호출한다. check가 성공하면 그 다음 깊이는 보지 않는다.

3. `itertools.combinations`으로 사다리를 뽑을 수 있다. [[풀이 - solve_itertools.py](./solve_itertools.py)]

    - 백트래킹 2번 방식처럼, 최소 사다리 개수를 0 ~ 3개로 해서 조합을 차례대로 뽑아본다.

    - 백트래킹보다 더 간결하고 직관적이다.

    - 인접한 사다리를 뽑은 경우를 미리 가지치기로 배제할 수 있다. (`check()` 호출 안하게)
        - 그러나, 실제로 가지치기를 하는게 더 효율적일지는 테스트 케이스에 따라 다르다.
        - 테스트 케이스에 인접한 경우가 많이 없는 경우, 인접한지 확인하는 비용이 `check()` 함수 호출하는 것보다 더 비쌀 수도 있다.
        - 이 문제의 경우, 가지치기를 하는 것이 시간이 더 오래 걸리는 것으로 채점되었다.
