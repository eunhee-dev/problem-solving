# 17837번. 새로운 게임 2

## 문제
https://www.acmicpc.net/problem/17837

---

## Key Points

1. 문제 지문이 길고 복잡해보이지만, 차근차근 지문대로 구현하면 되는 문제이다.

2. `pos_board`라는 3d 배열을 두어, 각 좌표에 말이 어떤 순서로 쌓여있는지 기록해두면 풀기 편하다.

    - 각 말의 좌표와 방향은 기존 `pieces` 배열에 기록한다.

    - `pos_board`는 각 좌표에 말이 어떤 순서로 쌓여있는지만 기록한다.

    - 말의 개수(K)가 10 이하로 작기 때문에, `pos_board.index(num)`을 통해 `num`번 말의 인덱스를 찾고, 그 위에 있는 말을 slicing을 활용하여 전부 옮겨준다.

3. 방향을 뒤집는 구현은 `DIRECTIONS.index((-dx, -dy))`을 활용하였다. (0 <-> 1, 2 <-> 3)

    - `(d // 2) * 2 + (d + 1) % 2`도 수학적으로 가능하지만, 위의 방식이 더 직관적이고, `DIRECTIONS` 순서가 바뀌어도 재사용 가능하다.

    - index 탐색에 O(4)가 발생하지만, 사실상 무시 가능한 수준이다.
