# 15898번. 피아의 아틀리에 ~신비한 대회의 연금술사~

## 문제
https://www.acmicpc.net/problem/15898

---

## Key Points

1. 문제를 다음과 같이 나누어 생각하면 좋다.

    - 순서를 고려하여 재료 3개 뽑기

    - 각 재료를 4번 회전시켜보기

    - (0, 0), (0, 1), (1, 0), (1, 1)을 시작점으로 4군데 배치시키기

2. `itertools.permutations`가 떠오르지만, 이전 단계까지의 연산을 그대로 활용하기 위해서는 백트래킹이 더 괜찮은 선택일 것 같다.

    - 3개를 뽑는거 까지만 `itertools.permutations`로 하고, 그 이후는 재귀나 다중 반복문으로하면 비슷한 효율성으로 구현이 가능하다.
        - 다만, 재귀를 사용할 경우 백트래킹 구조와 크게 다를 것이 없다. (오히려 복잡한 것 같다.)
        - 다중 반복문은 중복되는 코드가 많을 것 같아 나는 백트래킹을 선택하였다.

3. 파이썬으로 풀기 빡센 문제이다. 모든 경우를 시뮬레이션해도 통과할 것 같지만 파이썬은 그렇지 않다.

    - 재귀호출 횟수는 nP3(순서를 고려하여 3개 뽑기) * 4^3(4번 회전하여 배치) * 4^3((0, 0) ~ (1, 1) 4군데 배치)이다.

    - n이 10 이하이므로 통과할 수 있을 것 같아 보이는데, 시간 초과가 발생한다.
  
    - 어쩔 수 없이 최적화 할 수 있는 포인트를 찾아야 한다.

4. 최적화 포인트

    - (가장 중요) 첫번째 재료의 경우 4군데에 배치시키지 않아도 된다. [[풀이 - solve.py](./solve.py)]
        - 초기 가마는 5x5로 모두 품질이 0이고, W색이다. (상하좌우 대칭, 원점 대칭)
        - 재료를 4번 회전하기 때문에, (0, 0)에 배치시키는 경우만 고려해도 된다. (나머지는 결국 중복된다.)

    - 그 외의 최적화로 시간을 좀 더 단축시킬 수 있다. [[풀이 - solve_more_efficient.py](./solve_more_efficient.py)]
        - 재료를 미리 회전시켜놓는 방법
        - 품질을 마지막에 한 번에 계산하는 것이 아닌 미리 조금씩 계산해 놓는 방법
    
    - 최적화 할 수 있는 요소가 더 있을 것 같다. (CPython으로 통과할 수 있는 방법)
