# 17471번. 게리맨더링

## 문제
https://www.acmicpc.net/problem/17471

---

## Key Points

1. 구역들을 2개의 선거구로 나누고, 각 선거구가 서로 연결되어있는지 확인하여 연결되어 있다면 두 선거구의 인구차의 최소값을 구하면 되는 문제이다.

    - N이 10 이하이므로 완전 탐색이 가능하다.

    - 각 선거구가 연결되어 있는지 확인하는 것은 bfs를 통해 구현 가능하다.

2. 구역들을 2개의 선거구로 나누는 방법은 여러가지 방법이 있을 수 있다.

    - `itertools.combinations`를 활용하면 쉽게 선거구를 나눌 수 있다.
        - 가장 simple하고 가독성이 좋은 것 같다.

    - 백트래킹을 사용하여 선거구를 뽑는 방법이 있다.
        - 다만, 모든 경우의 수를 확인해봐야하기 때문에 가지치기를 할 여지가 없어 `itertools.combinations`에 비해 뚜렷한 장점이 없다.
        - 구현 상 약간의 시간 이득은 있을 수 있다.

    - dp를 활용하여 미리 계산해놓는 방법이 있다.
        - 구역들의 조합이 연결되어있는지, 각 구역 조합의 합을 미리 계산해 기록해놓을 수 있다. (비트마스킹을 활용하면 기록이 용이하다.)
        - 특히, 각 구역의 조합의 합을 구할 때, 매번 더하는 것이 아니라 기존 계산 값을 재사용할 수 있어 더 효율적이다. (dp 특성)

    - 구역의 조합은 전체 중 절반만 확인하면 된다. (A 그룹에 포함되지 않으면 B 그룹에 포함되기 때문에 대칭 구조이다.)

    - 구역의 조합을 모두 구하는데 2^N, 각 조합별로 bfs를 돌리는데 N^2, 총 시간복잡도는 O(2^N * N^2)으로 넉넉한 편이기에, 구현이 간단한 `itertools.combinatons`로 구현하였다.
